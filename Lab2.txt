Тема: Мобильное приложение "Organiser" (Ежедневник)
Цель лабораторной работы
Разработать приложение-органайзер на выбранной платформе.
При формировании и проектировании приложения необходимо соблюдать строгие требования, указанные ниже.
Дизайн приложения и выбор API/Framework остаются на личное усмотрение (в рамках лабораторной работы №2).

UI-Компоненты
Приложение должно содержать минимум 3 основных Activity, описанных ниже:

1. MainActivity
Calendar View (кастомный или стандартный)

Кнопки:

Добавить (Add)

Удалить (Remove)

Обновить (Update)

Поиск (по ключевым словам)

2. AddActivity
Контроллер даты/времени

Текстовое поле (для информации)

Кнопки (и другие элементы на усмотрение)

3. UpdateActivity
Фактически дублирует AddActivity, но с предзаполненными данными для редактирования.

Хранение данных
Все операционные данные должны храниться в XML-файле/файлах (формат на усмотрение).

Ключевые технологии:

Работа с XML

Сериализация данных

Логический/Операционный компонент
Все события, напоминания и уведомления (звуковые/визуальные) должны обрабатываться в отдельном сервисе.

Этот сервис будет извлекать данные из XML-файла и управлять уведомлениями.

Тема и функционал (на выбор)
Тема приложения может быть любой, но должна соответствовать его функционалу. Примеры:

Функционал	Тема приложения
Планировщик учебного расписания	Student Organiser
Напоминание о приёме лекарств	Medication Reminder
Личный ежедневник задач	Daily Task Manager
Важно: В отчете необходимо указать выбранную тему и её связь с функционалом.
